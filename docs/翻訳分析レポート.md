## Technical Notes

### Translation Key Format

All translation keys follow the pattern: `namespace.category.item`

Example: `common.buttons.signup` = Common namespace ‚Üí Buttons category ‚Üí Signup item

### Component Usage Pattern

Components should use the full namespaced key:

```typescript
// ‚úÖ Correct
const t = useTranslations()
t('common.buttons.signup')

// ‚ùå Incorrect (will cause MISSING_MESSAGE error)
const t = useTranslations()
t('buttons.signup')
```

### i18n Configuration

The `i18n.ts` file loads translations with namespace preservation:

```typescript
messages: {
  common: common.default,
  auth: auth.default,
  // ... other namespaces
}
```

This structure requires full namespace prefixes in all `t()` calls.
