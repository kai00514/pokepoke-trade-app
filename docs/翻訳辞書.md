  1. auth.json (Namespaced as auth.*)

  - login.* - Login page text (title, subtitle, signupPrompt, socialLoginNote)
  - signup.* - Signup page text (title, subtitle, emailFormTitle, termsAgreement, etc.)
  - reset.* - Password reset page (title, subtitle, newPasswordTitle, backToLogin)
  - modal.* - Auth modal content
  - loginPrompt.* - Login prompt text
  - accountBenefits, benefit1-5 - Account benefit descriptions
  - login, register, continueAsGuest - Action labels

  2. cards.json (Namespaced as cards.*)

  - categories.* - Card categories (all, pokemon, trainers, goods, tools)
  - types.* - Card types (grass, fire, water, electric, psychic, fighting, dark, steel, colorless, dragon)
  - rarities.* - Card rarities (diamond1-4, star1-3, crown, color1-2, with labels)
  - packs.* - Pack information
  - filters.* - Filter labels (type, pack)

  3. comments.json (Namespaced as comments.*)

  - Root level keys: title, titleWithCount, anonymousUser, guest, justNow, noComments
  - placeholder, placeholderGuest, pleaseEnter
  - posted, postedSuccess, postedAsGuest
  - postError, postErrorTitle, postErrorRetry, loadError
  - loggedInAs, edited, shortcutHint

  4. common.json (Namespaced as common.*)

  - navigation.* - Nav links (home, matching, decks, history, info, backToTimeline, backToTop, backToLists,
  backToLogin)
  - buttons.* - 40+ button labels (signup, login, logout, createTrade, generateCollage, createCardList, etc.)
  - labels.* - UI labels (or, soon, recruiting, inProgress, completed, negotiable, user, adSpace,
  profileBanner)
  - header.* - Header text (openUserMenu, userAvatar, logoAlt, notAuthenticated)
  - footer.* - Footer text
  - misc.* - Misc UI (selectLanguage, confirm, switchLanguage, notSelected, noCardsYet, processing, loading)
  - user.* - User labels (online, setYourId, customizeDisplayName)
  - contact.* - Contact prompts
  - auth.* - Auth text (safeLogout)
  - language.* - Language names (ja, en, zh-CN, zh-TW, ko, pt-BR, fr, it, de, es)
  - notifications.* - Notification system (title, unreadCount, markAsRead, types., time.)

  5. decks.json (Namespaced as decks.*)

  - Root level: untitled, noCards, noRecipe, updated, type, commentCount, cardListLabel
  - categories.* - Deck categories (all, pokemon, trainers, goods, tools, tier, featured, newPack, post)

  6. errors.json (Namespaced as errors.*)

  - Root level: title, inputError, unexpected, operationFailed, loginRequired
  - auth.* - Auth errors (generic, callbackError, noCode, invalidCredentials, emailNotConfirmed, etc.)
  - validation.* - Form validation errors (emptyTitle, noWantedCards, requiredFields, maxSelection)
  - trade.* - Trade errors (inputError, sessionError, postCreationFailed)
  - data.* - Data fetch errors (fetchError, postsFetchFailed, packsFetchError)
  - list.* - List errors (authRequired, creationError, creationFailed)
  - share.* - Share errors (copyFailed, tryAgain)
  - collage.* - Collage errors (emptyTitle, noCards, generationFailed)
  - profile.* - Profile errors (fetchFailed, fetchFailedMessage)

  7. evaluation.json (Namespaced as evaluation.*)

  - Root level: checkError, checkErrorDetail, loginPrompt, alreadyRated, alreadyRatedDetail
  - submitError, submitErrorDetail, submitSuccess, submitSuccessDetail, deckFeatures
  - stats.* - Evaluation stats (accessibility, speed, power, durability, stability)

  8. forms.json (Namespaced as forms.*)

  - auth.* - Auth forms (email., password., confirmPassword., newPassword.)
  - trade.* - Trade forms (title., wantedCards., offeredCards., savedList., pokepokeId., comment.)
  - list.* - List forms (name., description.)
  - search.* - Search forms (keyword.*)
  - collage.* - Collage forms (group1Title., group2Title.)
  - contact.* - Contact forms (name., email., subject., message.)
  - placeholders.* - Form placeholders (keyword, enterListName)

  9. messages.json (Namespaced as messages.*)

  - success.* - Success messages (loginSuccess, tradePosted, listCreated, urlCopied, etc.)
  - info.* - Info messages (tradeNotice, collageDescription, cardsSelected, noCardsFound, etc.)
  - warning.* - Warning messages (timeSkew, selectionLimit)

  10. pages.json (Namespaced as pages.*)

  - home.* - Homepage (title, subtitle)
  - login.*, signup.*, reset.* - Auth pages
  - createTrade.* - Trade creation (title, noticeTitle)
  - createList.* - List creation (title, subtitle, listInfo., cardSelection., loginRequired.*)
  - collages.* - Collages page (title, subtitle)
  - contact.* - Contact page (title, subtitle, form., faq., contactInfo., notice.)
  - tradePost.* - Trade post (wantedCards, offeredCards, authorComment)
  - share.* - Share modal (OBJECT with title, subtitle, urlSection, snsSection, tip)
  - detailedSearch.* - Search page (title)
  - collageGenerator.* - Collage tool (title, defaultTitle1, defaultTitle2, editGroup)
  - matching.* - Matching page (title, comingSoon, underDevelopment, etc.)
  - history.* - History page (title, myPosts, commentedPosts, matchingHistory, etc.)
  - decks.* - Decks page (postDeck, favorites, tierDecks, etc.)
  - resultsInfo.* - Results display (showing, searchResults, noPosts, noResults)

  11. status.json (Namespaced as status.*)

  - Root level: recruiting, inProgress, completed, completedShort, canceled

  12. survey.json (Namespaced as survey.*)

  - Root level: title, subtitle, incompleteError, submitError, thankYou, thankYouMessage
  - q1.*, q2.*, q3.*, q4.* - Survey questions

  13. trades.json (Namespaced as trades.*)

  - Root level: wantedCards, offeredCards, offeredCardsAlt, noCards, ogTitle
  - confirmAction, actionCompleted, statusChanged, actionFailed, actionError
  - ownerActions, complete, authorComment, authorCommentWithText, idCopied, notApplicable

  ---
  Current i18n.ts Configuration: All files are namespaced, so component calls should use format like:
  - t("common.buttons.signup")
  - t("pages.home.title")
  - t("errors.auth.invalidCredentials")
  - t("messages.success.loginSuccess")

  Note: pages.share is an object, not a string, so calling t("share") directly will cause the
  INSUFFICIENT_PATH error. Components should use t("common.buttons.share") for the button label instead.
